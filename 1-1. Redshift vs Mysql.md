# Redshift → MySQL 비호환 함수 치환표

> Redshift(PostgreSQL 기반)에서 쓰던 함수 중 **MySQL에서 안 되거나 다른 것**만 정리했어요. 동일하게 동작하는 함수는 생략합니다.

---

## 시험 직전 암기 — 핵심 10개

|Redshift|MySQL|비고|
|---|---|---|
|`DATEDIFF(day, start, end)`|`DATEDIFF(end, start)`|**인자 순서 반대!** MySQL은 day 단위만|
|`DATEADD(day, 7, d)`|`DATE_ADD(d, INTERVAL 7 DAY)`|또는 `d + INTERVAL 7 DAY`|
|`DATE_TRUNC('month', d)`|`DATE_FORMAT(d, '%Y-%m-01')`|MySQL에 DATE_TRUNC 없음|
|`GETDATE()` / `SYSDATE`|`NOW()` / `CURRENT_TIMESTAMP`||
|`NVL(a, b)`|`IFNULL(a, b)` 또는 `COALESCE(a, b)`||
|`LEN(s)`|`CHAR_LENGTH(s)`|`LENGTH()`는 바이트 수 (한글 주의)|
|`LISTAGG(col, ',') WITHIN GROUP (ORDER BY col)`|`GROUP_CONCAT(col ORDER BY col SEPARATOR ',')`||
|`CONVERT_TIMEZONE('UTC','Asia/Seoul',ts)`|`CONVERT_TZ(ts, 'UTC', 'Asia/Seoul')`||
|`::int`, `::text` (캐스팅)|`CAST(x AS SIGNED)`, `CAST(x AS CHAR)`|MySQL은 `::` 문법 없음|
|`QUALIFY ROW_NUMBER() OVER(...) = 1`|서브쿼리/CTE로 우회|MySQL에도 QUALIFY 없음|

## 날짜/시간

|용도|Redshift|MySQL|
|---|---|---|
|날짜 차이 (일)|`DATEDIFF(day, start, end)`|`DATEDIFF(end, start)` ← 순서 반대, day만 가능|
|날짜 차이 (시간/분/초)|`DATEDIFF(hour, start, end)`|`TIMESTAMPDIFF(HOUR, start, end)`|
|날짜 더하기|`DATEADD(month, 3, d)`|`DATE_ADD(d, INTERVAL 3 MONTH)`|
|날짜 빼기|`DATEADD(day, -30, d)`|`DATE_SUB(d, INTERVAL 30 DAY)` 또는 `d - INTERVAL 30 DAY`|
|월 시작일|`DATE_TRUNC('month', d)`|`DATE_FORMAT(d, '%Y-%m-01')` 또는 `LAST_DAY(d) - INTERVAL (DAY(d)-1) DAY`|
|주 시작일|`DATE_TRUNC('week', d)`|`DATE_SUB(d, INTERVAL WEEKDAY(d) DAY)` (월요일 기준)|
|현재 날짜|`CURRENT_DATE`|`CURRENT_DATE` 또는 `CURDATE()`|
|현재 시각|`GETDATE()`|`NOW()`|
|타임스탬프→날짜|`d::date` 또는 `DATE(ts)`|`DATE(ts)`|
|요일 추출|`EXTRACT(DOW FROM d)` (0=일)|`DAYOFWEEK(d)` (1=일) — **시작 번호 다름**|
|날짜 시퀀스 생성|없음 (generate_series는 PG만)|없음 (재귀 CTE나 numbers 테이블 활용)|

**`DATEDIFF` 인자 순서가 반대인 거 진짜 조심하세요.** Redshift는 `DATEDIFF(unit, start, end)`, MySQL은 `DATEDIFF(end, start)` — 습관적으로 Redshift 순서로 쓰면 부호가 뒤집혀요.

**MySQL의 `TIMESTAMPDIFF`는 Redshift `DATEDIFF`의 완전한 대응:**

|Redshift|MySQL|
|---|---|
|`DATEDIFF(day, a, b)`|`TIMESTAMPDIFF(DAY, a, b)`|
|`DATEDIFF(hour, a, b)`|`TIMESTAMPDIFF(HOUR, a, b)`|
|`DATEDIFF(month, a, b)`|`TIMESTAMPDIFF(MONTH, a, b)`|
|`DATEDIFF(year, a, b)`|`TIMESTAMPDIFF(YEAR, a, b)`|

`TIMESTAMPDIFF`는 인자 순서가 Redshift와 같아요 `(unit, start, end)`. **`DATEDIFF`보다 `TIMESTAMPDIFF`를 쓰는 게 안전합니다.**

## 문자열

|용도|Redshift|MySQL|
|---|---|---|
|길이|`LEN(s)`|`CHAR_LENGTH(s)` — `LENGTH()`는 바이트라 한글 3배|
|위치 찾기|`CHARINDEX(sub, s)` 또는 `POSITION(sub IN s)`|`LOCATE(sub, s)` 또는 `INSTR(s, sub)` — **인자 순서 주의**|
|그룹 문자열 집계|`LISTAGG(col, ',') WITHIN GROUP (ORDER BY col)`|`GROUP_CONCAT(col ORDER BY col SEPARATOR ',')`|
|정규식 매칭|`col ~ pattern`|`col REGEXP pattern`|
|정규식 추출|`REGEXP_SUBSTR(s, pattern)`|`REGEXP_SUBSTR(s, pattern)` (MySQL 8.0+)|
|정규식 치환|`REGEXP_REPLACE(s, pattern, repl)`|`REGEXP_REPLACE(s, pattern, repl)` (MySQL 8.0+)|
|문자열 합치기 + NULL|`a \| b` (NULL이면 NULL)|`CONCAT(a, b)` (NULL을 '' 취급)|

## NULL 처리

|용도|Redshift|MySQL|
|---|---|---|
|NULL → 기본값|`NVL(a, b)`|`IFNULL(a, b)` 또는 `COALESCE(a, b)`|
|같으면 NULL|`NULLIF(a, b)`|`NULLIF(a, b)` ✅|
|NULL이면 다른 값|`COALESCE(a, b, c)`|`COALESCE(a, b, c)` ✅|

## 타입 캐스팅

|Redshift|MySQL|
|---|---|
|`col::int`|`CAST(col AS SIGNED)` — MySQL에 `::` 문법 없음|
|`col::varchar`|`CAST(col AS CHAR)`|
|`col::float`|`CAST(col AS DOUBLE)` 또는 `col * 1.0`|
|`col::date`|`CAST(col AS DATE)` 또는 `DATE(col)`|
|`col::timestamp`|`CAST(col AS DATETIME)`|

**MySQL 정수 타입명 주의**: `INTEGER`가 아니라 `SIGNED` 또는 `UNSIGNED`로 캐스팅.

## 윈도우 함수

MySQL 8.0+에서 윈도우 함수 대부분 지원. **거의 동일하게 써도 돼요.** 단:

|이슈|상세|
|---|---|
|`QUALIFY`|MySQL 없음 → CTE + WHERE로 우회|
|`PERCENTILE_CONT`|MySQL 없음 → `PERCENT_RANK` + 서브쿼리로 우회하거나 변수 활용|
|`NTH_VALUE`|MySQL 8.0+ 지원 ✅|
|`ROWS BETWEEN ... RANGE BETWEEN`|MySQL 8.0+ 지원 ✅|

---

## 집계 / 그룹

|용도|Redshift|MySQL|
|---|---|---|
|조건부 집계 (`FILTER` 절)|`CASE WHEN` (Redshift도 FILTER 없음)|`CASE WHEN` (양쪽 동일)|
|`GROUPING SETS`|✅ 지원|❌ 미지원 — `UNION ALL`로 우회|
|`ROLLUP`|`GROUP BY ROLLUP(a, b)`|`GROUP BY a, b WITH ROLLUP` — **문법 다름**|
|`CUBE`|✅ 지원|❌ 미지원|
|`GROUPING(col)`|✅ 지원|`GROUPING(col)` (MySQL 8.0+, ROLLUP에서만)|

## 기타 주의

|이슈|Redshift|MySQL|
|---|---|---|
|`LIMIT + OFFSET`|`LIMIT n OFFSET m`|`LIMIT m, n` 또는 `LIMIT n OFFSET m` — 순서 주의|
|`EXCEPT`|✅|`EXCEPT` (MySQL 8.0.31+) — 이전 버전은 `NOT IN`/`NOT EXISTS`로|
|`INTERSECT`|✅|MySQL 8.0.31+|
|`BOOLEAN`|`TRUE/FALSE`|`TRUE/FALSE` (내부적으로 1/0)|
|CTE (WITH)|✅|MySQL 8.0+ ✅|
|재귀 CTE|✅|`WITH RECURSIVE` (RECURSIVE 키워드 필수)|
|`ILIKE` (대소문자 무시)|✅|MySQL은 기본이 case-insensitive (`LIKE`가 이미 무시)|
|`DISTINCT ON`|✅|❌ — `GROUP BY` + 집계 또는 윈도우로 우회|

## 면접 직전 최종 체크

```
DATEDIFF(day, a, b)  →  TIMESTAMPDIFF(DAY, a, b)     ← 이름+인자순서 주의
DATEADD(day, 7, d)   →  DATE_ADD(d, INTERVAL 7 DAY)
DATE_TRUNC('month')  →  DATE_FORMAT(d, '%Y-%m-01')
GETDATE() / SYSDATE  →  NOW()
NVL(a, b)            →  IFNULL(a, b) 또는 COALESCE
LEN(s)               →  CHAR_LENGTH(s)
LISTAGG              →  GROUP_CONCAT
col::int             →  CAST(col AS SIGNED)
QUALIFY              →  CTE + WHERE
ILIKE                →  LIKE (MySQL 기본이 case-insensitive)
```
